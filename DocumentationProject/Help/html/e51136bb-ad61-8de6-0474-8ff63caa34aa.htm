Received0 Indicates that a request has been received and has not begun processing.

Requests in this state have not undergone validation of any type and assumptions regarding their contained data should not be made.

Processed1 Indicates a message has been processed successfully prior to authentication or method invocation.
Authenticated2 Inidicates a message has been successfully authenticated prior to method invocation but after message processing.
Invoked3 Indicates that the message has completed invoking its endpoint's data handler. The final actions are final logging and response release.
Handled4 Indicates that the message has been processed, possible authenticated, and has passed its data handler.

Additionally, the request has completed final logging and is only waiting to release the response.

At this point the message is complete and the response need only be sent back to the client.

FailedAtProcessing5 Indicates the message failed during the processing step.
FailedAtAuthentication6 Indicates the message failed during the authentication step.
FailedAtInvocation7 Indicates the message failed during the invocation step.
FailedAtFinalHandling8 Indicates the message failed during the final handling step.
